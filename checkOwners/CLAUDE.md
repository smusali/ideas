# checkOwners

CODEOWNERS inference engine driven by git commit history with drift detection and CI integration.

## Stack

- Python 3.11+
- CLI: Typer + Rich (colored output, progress bars)
- Git analysis: GitPython + subprocess (`git log`, `git blame`)
- GitHub integration: PyGithub (`GITHUB_OUTPUT` for Actions)
- Config: PyYAML (`.github/checkowners.yml`)
- Packaging: hatch (PyPI); composite GitHub Action (`action.yml`)
- Testing: pytest + pytest-cov

## Commands

- `hatch run test`: run pytest suite with coverage
- `hatch run lint`: ruff check + mypy
- `hatch run fmt`: ruff format
- `checkowners analyze`: infer ownership from git history
- `checkowners generate`: write `.github/CODEOWNERS`
- `checkowners print`: print inferred owners to stdout
- `checkowners validate`: validate existing CODEOWNERS syntax only
- `checkowners drift`: compare inferred vs. current CODEOWNERS
- `checkowners notify`: POST webhook on drift
- `checkowners sync`: generate + commit CODEOWNERS

## Architecture

```
checkowners/
  cli.py        # Typer app; all subcommands registered here
  analyze.py    # git log + git blame -> OwnershipMap
  generate.py   # OwnershipMap -> .github/CODEOWNERS writer
  drift.py      # State machine: inferred vs. current diff -> DriftResult
  notify.py     # Webhook POST on drift events
  validate.py   # Syntax-only CODEOWNERS validator
  config.py     # PyYAML loader for .github/checkowners.yml
  models.py     # Dataclasses: OwnershipMap, DriftResult, Config
  state.py      # ~/.checkowners/state.json reader/writer
tests/
  test_analyze.py
  test_generate.py
  test_drift.py
  test_validate.py
  integration/  # Fixture git repos via GitPython
action.yml      # Composite GitHub Action
pyproject.toml  # hatch project config
CODEOWNERS      # Auto-maintained by checkowners generate
```

## Conventions

- Functional style throughout; no classes except dataclasses in `models.py`
- Type hints on every function signature; strict mypy (`--strict`)
- All CLI commands support `--json` flag for structured JSON output
- State file: `~/.checkowners/state.json` (auto-maintained, never hardcode path)
- Config file: `.github/checkowners.yml` (per-repo; loaded via `config.py`)
- Every write to `.github/CODEOWNERS` must include the machine-generated header:
  `# Generated by checkOwners. Do not edit manually.`
- Default exclusions: `*.lock`, `dist/**`, `vendor/**`
- Drift state machine modes: `commit`, `repo`, `both`
- Config defaults: `lookback_days: 180`, `min_commits: 3`, `top_n_owners: 2`

## Testing

- Every module has a corresponding test file in `tests/`
- Unit tests mock all subprocess calls (`git log`, `git blame`); never require a real git repo
- Integration tests in `tests/integration/` use fixture repos created with GitPython
- Coverage target: 85%+ on every module
- Run `hatch run test` before every commit; block merges below coverage threshold

## Do NOT

- No LLM or AI dependencies; checkOwners is pure-git inference only
- No write to `.github/CODEOWNERS` without the machine-generated header
- No external network calls except in `notify.py` (webhook POST) and `action.yml` (GitHub API)
- No classes except dataclasses in `models.py`
- No skipped or missing type annotations
- No hardcoded file paths; use `pathlib.Path` and config-driven paths everywhere
- No modification of `state.json` schema without bumping the schema version field
